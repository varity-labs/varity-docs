## Quick Example

The most common pattern is the zero-config database:

```typescript
import { db } from '@varity-labs/sdk';

// Define a collection with TypeScript types
const users = db.collection<{ name: string; email: string }>('users');

// CRUD operations
await users.add({ name: 'Alice', email: 'alice@example.com' });
const allUsers = await users.get({ limit: 10, orderBy: 'name' });
await users.update('user-id', { name: 'Alice Smith' });
await users.delete('user-id');
```

For apps that need multiple modules (analytics, storage, auth), use the full SDK:

```typescript
import { createVaritySDK } from '@varity-labs/sdk';

const sdk = createVaritySDK({
  network: 'arbitrum-l3-testnet',
});

await sdk.connect();

const kpis = await sdk.analytics.getKPIs({ period: 'current_month' });
const stored = await sdk.storage.upload(fileData);
```

## Core Exports

The SDK exports several categories of functionality.

### SDK Core

| Export | Description |
|--------|-------------|
| `VaritySDK` | Main SDK class |
| `createVaritySDK` | Factory function to create SDK instance |
| `db` | Zero-config database (recommended for most apps) |
| `Database` | Database class |
| `Collection` | Collection class for typed data access |
| `getNetworkConfig` | Get network configuration by name |
| `NETWORK_CONFIGS` | All supported network configurations |

### Development Credentials

| Export | Description |
|--------|-------------|
| `VARITY_DEV_CREDENTIALS` | Shared dev credentials for quick start |
| `resolveCredentials` | Resolve with fallback to dev credentials |
| `isUsingDevCredentials` | Check if using shared credentials |

## Capability Modules

The SDK provides 13 capability modules that work across all application types.

| Module | Purpose |
|--------|---------|
| `sdk.auth` | Authentication and access control |
| `sdk.storage` | File storage |
| `sdk.db` | Database operations (also available standalone: `import { db } from '@varity-labs/sdk'`) |
| `sdk.s3` | S3-compatible storage (optional) |
| `sdk.compute` | AI/LLM queries |
| `sdk.contracts` | Backend interactions |
| `sdk.analytics` | KPIs, trends, growth metrics |
| `sdk.notifications` | Email, push, webhook notifications |
| `sdk.export` | CSV, JSON, PDF exports |
| `sdk.cache` | In-memory caching |
| `sdk.monitoring` | Health checks and metrics |
| `sdk.forecasting` | Predictive analytics |
| `sdk.webhooks` | Webhook management |

## Complete Working Example

```typescript
import { db, createVaritySDK } from '@varity-labs/sdk';

// Zero-config database
const products = db.collection<{ name: string; price: number }>('products');
await products.add({ name: 'Pro Plan', price: 99 });
const allProducts = await products.get();

// Full SDK for multi-module access
const sdk = createVaritySDK({
  network: 'arbitrum-l3-testnet',
});

await sdk.connect();

// Upload a file
const result = await sdk.storage.upload(myFile);
console.log('File URL:', result.url);

// Get analytics
const kpis = await sdk.analytics.getKPIs({ period: 'current_month' });
console.log('Total users:', kpis.totalUsers);
```

For power users who need lower-level access to Varity's infrastructure, the SDK provides subpath imports:

### Infrastructure Configuration

Chain configuration and utilities are available via subpath imports:

```typescript
// Infrastructure configuration (not in main entry)
import { varityL3Testnet, formatUSDC, parseUSDC } from '@varity-labs/sdk/chains';
```

### Contract Interaction

```typescript
import { createThirdwebClient, getContract, readContract } from 'thirdweb';
import { varityL3Testnet } from '@varity-labs/sdk/chains';

const client = createThirdwebClient({
  clientId: process.env.NEXT_PUBLIC_THIRDWEB_CLIENT_ID,
});

const contract = getContract({
  client,
  chain: varityL3Testnet,
  address: '0x58618dcD2A4109854441B74a28C1dA4E58d7AF44', // TemplateMarketplace
});

const priceRaw = await readContract({
  contract,
  method: 'function getPrice(uint256) view returns (uint256)',
  params: [BigInt('1')],
});
```

### Engine Client

| Client | Purpose |
|--------|---------|
| `EngineClient` | Production operation management |
| `NebulaClient` | AI-powered interactions |
| `StorageClient` | File storage |
| `BridgeClient` | Cross-chain transfers |
| `GatewayClient` | RPC infrastructure |

```typescript
import { createEngineClient } from '@varity-labs/sdk';

const engine = createEngineClient({
  engineUrl: process.env.THIRDWEB_ENGINE_URL,
  accessToken: process.env.THIRDWEB_ACCESS_TOKEN,
  backendWallet: process.env.BACKEND_WALLET_ADDRESS,
});

const result = await engine.sendTransaction({
  chainId: 33529,
  to: contractAddress,
  data: txData,
});
```
