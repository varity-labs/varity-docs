Get the VarityKit CLI installed on your system.

## Install with pip

<Tabs>
  <TabItem label="pip">
    ```bash
    pip install varitykit
    ```
  </TabItem>
  <TabItem label="pipx (Recommended)">
    ```bash
    pipx install varitykit
    ```
  </TabItem>
  <TabItem label="From Source">
    ```bash
    git clone https://github.com/varity-labs/varity-sdk.git
    cd varity-sdk/cli
    pip install -e .
    ```
  </TabItem>
</Tabs>

<Aside type="tip">
**pipx** is recommended for CLI tools because it installs packages in isolated environments to avoid dependency conflicts.
</Aside>

## Verify Installation

```bash
varitykit --version
```

You should see the current version number, for example:

```
varitykit 1.2.0
```

## Run Diagnostics

Check that your environment is properly configured:

```bash
varitykit doctor
```

You should see a table of checks with a `✓ All checks passed!` message at the end.

## Environment Setup

<Steps>

1. **Create a .env file** (optional) in your project root:

   ```bash title=".env.local"
   # Your Varity App ID (from developer portal)
   NEXT_PUBLIC_VARITY_APP_ID=your-app-id
   ```

   All deployment credentials (storage, authentication, hosting) are managed automatically by the Varity credential proxy. See [Managed Credentials](/deploy/managed-credentials).

2. **Verify your setup**:

   ```bash
   varitykit doctor
   ```

</Steps>

## Shell Completions

Enable tab completion for your shell:

```bash
# Auto-detect shell and install
varitykit completions --install

# Or specify a shell
varitykit completions --shell bash --install
```

Run `varitykit completions` to see manual installation instructions for your shell.

## Troubleshooting

### "command not found: varitykit"

The CLI is not in your PATH. Try:

```bash
# Option 1: Use python -m
python -m varitykit --version

# Option 2: Add to PATH
export PATH="$HOME/.local/bin:$PATH"
```

### "ModuleNotFoundError"

Python dependencies are missing:

```bash
pip install --upgrade varitykit
```

### "Docker not found"

Docker is optional and used for advanced local development:

```bash
# macOS
brew install --cask docker

# Ubuntu
sudo apt install docker.io

# Windows
# Download from docker.com
```

### Permission denied

On Linux/macOS, you may need to run:

```bash
pip install --user varitykit
```

Or use a virtual environment:

```bash
python -m venv .venv
source .venv/bin/activate
pip install varitykit
```

## Updating

Update to the latest version:

```bash
pip install --upgrade varitykit
```

## Uninstalling

Remove the CLI:

```bash
pip uninstall varitykit
```
