## 2. Callout Component

Four semantic types for different message priorities.

<Callout type="tip" title="Pro tip">
  Use environment variables for all API keys and sensitive configuration.
  Never commit credentials to Git.
</Callout>

<Callout type="note">
  The default RPC endpoint is `https://rpc-varity-testnet.t.conduit.xyz`.
  You can override this in your Varity SDK configuration.
</Callout>

<Callout type="warning" title="Breaking change in v2.0">
  The SDK now uses a modular pattern. Use `db` for database operations
  and standalone functions for credentials instead of a client instance.
</Callout>

<Callout type="danger" title="Security warning">
  Never expose your service role key in client-side code. This key has
  full access to your Varity project and should only be used in server
  environments.
</Callout>

### Collapsible Callout

<Callout type="tip" title="Advanced usage" collapsible={true} defaultOpen={false}>
  You can pass additional options to customize the database:

  ```typescript
  import { Database } from '@varity-labs/sdk'

  const customDb = new Database({
    proxyUrl: 'https://your-proxy.example.com',
    appToken: 'your-app-token',
  })
  ```

  See the [SDK Overview](/packages/sdk/overview) for all options.
</Callout>

## 6. Accordion Component

Collapsible content for advanced/optional sections.

### Default Variant

<Accordion title="What is Varity infrastructure?">
  Varity infrastructure powers your applications.
  It provides:

  - **Free operations** for users (sponsored by developers)
  - **Sub-second finality** for fast responses
  - **Full compatibility** with existing development tools
  - **Usage-based pricing** for developers
</Accordion>

### Advanced Variant

<Accordion title="Advanced: How credential provisioning works" variant="advanced">
  Varity automatically provisions and rotates credentials for your application.
  When a user signs in:

  1. **Authentication** verifies the user's identity via email or social login
  2. **Session management** creates a secure, short-lived session
  3. **Credential provisioning** grants scoped access to the user's data
  4. **Automatic rotation** keeps credentials fresh without user action

  All credential management happens automatically — developers use the
  authentication hooks from `@varity-labs/ui-kit`.

  [Learn more about authentication](/build/auth/quickstart)
</Accordion>

### Example Variant

<Accordion title="Example: Complete authentication flow" variant="example" defaultOpen={true}>
  Here's a complete example showing login, user state, and logout:

  ```tsx
  import { AuthProvider, LoginButton, useAuth } from '@varity-labs/ui-kit'

  // Set up auth provider
  function App() {
    return (
      <AuthProvider appId={process.env.NEXT_PUBLIC_VARITY_AUTH_APP_ID}>
        <AuthExample />
      </AuthProvider>
    )
  }

  // Use the auth hook for state
  function AuthExample() {
    const { ready, authenticated, user, logout } = useAuth()

    if (!ready) return <p>Loading...</p>

    if (!authenticated) {
      return <LoginButton />
    }

    return (
      <div>
        <p>Logged in as: {user?.email?.address}</p>
        <button onClick={logout}>Log out</button>
      </div>
    )
  }
  ```
</Accordion>
