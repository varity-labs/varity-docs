Deploy your application to Varity's secure infrastructure.

## Usage

```bash
varitykit app deploy [OPTIONS]
```

## Hosting Options

### Static Sites

Deploy static websites and single-page applications:

```bash
varitykit app deploy
```

**Best for:**
- Marketing websites
- Documentation sites
- Single-page applications
- Landing pages

**Features:**
- Global CDN distribution
- Permanent deployment URLs
- Automatic SSL
- One flat monthly cost per app, your bill does not change with traffic

### Dynamic Apps

Deploy full-stack applications with compute resources:

```bash
varitykit app deploy --hosting dynamic
```

**Best for:**
- APIs and backends
- Server-side rendering
- Database applications
- Full-stack apps

**Features:**
- Container hosting
- Persistent storage
- One flat monthly cost per app. Your bill does not change with traffic

<Aside type="tip">
Dynamic hosting is powered by cloud compute. Use `--hosting dynamic` for server-side frameworks (Next.js API routes, Express, etc.). Static hosting remains the default.
</Aside>

## Options

| Option | Default | Description |
|--------|---------|-------------|
| `--hosting` | `auto` | Hosting type: `auto`, `static`, or `dynamic` |
| `--path` | `.` | Project directory |
| `--name` | None | Custom app name. Static apps live at `https://varity.app/<name>/`; dynamic apps live at `https://<name>.varity.app/` |
| `--mode` | `auto` | Deployment mode (advanced) |
| `--target` | None | Deployment target |
| `--tier` | None | Hardware tier to reserve |
| `--repo-url` | None | Deploy directly from a GitHub repo URL |
| `--skip-build` | `false` | Skip the build step (deploy prebuilt output) |
| `--image` | None | Deploy a prebuilt OCI image instead of building from source |
| `--image-registry` | None | Registry host for a private `--image` |
| `--image-username` | None | Username for a private `--image` |
| `--image-password` | None | Password or secret value for a private `--image` |
| `--port` | `80` | Port your dynamic app listens on |
| `--dry-run` | `false` | Preview the deployment without shipping (not compatible with `--image`) |

## Basic Deployment

<Steps>

1. **Build your project**

   ```bash
   npm run build
   ```

2. **Deploy**

   ```bash
   varitykit app deploy
   ```

3. **View your live app**

   ```
   ✓ Build complete
   ✓ Deployed successfully
   ✓ Registered on Varity

   Your app is live at:
   https://varity.app/your-app/

   Deployment ID: deploy-1737492000
   ```

</Steps>

<Aside type="note">
Static apps live at a path: `https://varity.app/<name>/`. Dynamic apps get their own subdomain: `https://<name>.varity.app/` (the apex path redirects to the subdomain).
</Aside>

## Supported Frameworks

The CLI automatically detects your framework:

| Framework | Detection | Build Command |
|-----------|-----------|---------------|
| Next.js | `next.config.*` | `npm run build` |
| React (Vite) | `vite.config.*` | `npm run build` |
| React (CRA) | `react-scripts` | `npm run build` |
| Vue | `vue.config.*` | `npm run build` |
| Custom | `package.json` | `npm run build` |

## Environment Variables

Storage and service credentials are auto-provided by Varity. No manual credential setup is required. See [Managed Credentials](/deploy/managed-credentials).

## List Deployments

View all deployments:

```bash
varitykit app list
```

Output:

```
ID                  | Status  | URL
--------------------|---------|---------------------------
deploy-1737492000   | live    | https://varity.app/your-app/
deploy-1737491000   | archived| https://varity.app/your-app/ (archived)
```

## View Deployment Details

Get detailed information about a specific deployment:

```bash
varitykit app info deploy-1737492000
```

## Delete a Deployment

Remove a deployment:

```bash
varitykit app delete deploy-1737492000 --yes
```

## Troubleshooting

### "Build failed"

1. Verify your build works locally:
   ```bash
   npm run build
   ```

2. Check for missing dependencies:
   ```bash
   npm install
   ```

### "Upload failed"

1. Verify network connectivity:
   ```bash
   varitykit doctor
   ```

2. Check your internet connection and try again
