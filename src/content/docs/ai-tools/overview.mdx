# AI Tools for Varity

Varity provides first-class AI tooling to help developers build faster and more efficiently. Our AI-native approach includes an MCP server for Claude, Cursor rules for intelligent code completion, and LLM-optimized documentation.

## Available AI Tools

### 1. Varity MCP Server

The Model Context Protocol (MCP) server allows AI assistants like Claude Desktop to interact with your Varity projects programmatically.

**Features:**
- Deploy applications directly from Claude
- Search Varity documentation
- Get project metrics and logs
- Execute CLI commands
- Debug deployment issues

**Setup:**

Add to your Claude Desktop configuration (`~/Library/Application Support/Claude/config.json` on Mac):

```json
{
  "mcpServers": {
    "varity": {
      "type": "http",
      "url": "https://mcp.varity.dev/mcp",
      "options": {
        "project_ref": "your-project-id",
        "read_only": false,
        "features": "deployment,docs,debugging"
      }
    }
  }
}
```

[Learn more about the MCP Server →](/ai-tools/mcp-server-spec)

### 2. Cursor Rules

Pre-configured AI rules for Cursor IDE that provide context-aware assistance for Varity development.

**Available Rules:**

1. **Deploy Applications** - Best practices for deploying to Varity
2. **Authentication Setup** - Implement social login and wallet auth
3. **Storage Configuration** - File upload and storage patterns
4. **Error Handling** - Robust error handling strategies
5. **Testing** - Testing patterns and best practices
6. **TypeScript** - Type-safe Varity development
7. **Performance** - Optimization techniques
8. **Backend Services** (Advanced) - Infrastructure integration

**Quick Install:**

```bash
curl -fsSL https://docs.varity.so/cursor-rules/install-varity-rules.sh | bash
```

**Manual Install:**

```bash
mkdir -p .cursor/rules
curl -o .cursor/rules/varity-deploy.mdc https://docs.varity.so/cursor-rules/varity-deploy.mdc
curl -o .cursor/rules/varity-auth.mdc https://docs.varity.so/cursor-rules/varity-auth.mdc
curl -o .cursor/rules/varity-storage.mdc https://docs.varity.so/cursor-rules/varity-storage.mdc
curl -o .cursor/rules/varity-errors.mdc https://docs.varity.so/cursor-rules/varity-errors.mdc
curl -o .cursor/rules/varity-testing.mdc https://docs.varity.so/cursor-rules/varity-testing.mdc
curl -o .cursor/rules/varity-typescript.mdc https://docs.varity.so/cursor-rules/varity-typescript.mdc
curl -o .cursor/rules/varity-performance.mdc https://docs.varity.so/cursor-rules/varity-performance.mdc
```

## AI-Friendly Code Patterns

Our AI tools are trained on these recommended patterns:

### Environment Variables

Always use the `<SUBSTITUTE_*>` format for documentation:

```bash
VARITY_URL=<SUBSTITUTE_VARITY_URL>
VARITY_KEY=<SUBSTITUTE_VARITY_KEY>
```

### Error Handling

Use error codes, not messages:

```typescript
try {
  await varity.storage.upload(file)
} catch (error) {
  if (error.code === 'FILE_TOO_LARGE') {
    // Handle specific error
  }
}
```

### Type Safety

Import types from `@varity-labs/types`:

```typescript
import type { UploadResult, VarityError } from '@varity-labs/types'
```

### Infrastructure Abstraction

AI assistants are trained to avoid technical infrastructure jargon:

❌ "Connect your wallet to deploy smart contracts on-chain"
✅ "Sign in to deploy your application to Varity"

## Advanced: Custom AI Integration

### Build Your Own MCP Tools

You can extend the Varity MCP server with custom tools:

```typescript
// custom-mcp-tool.ts
export async function customDeployTool(params: DeployParams) {
  // Your custom deployment logic
  const result = await varitykit.deploy(params)
  return result
}
```

### Create Custom Cursor Rules

Write your own `.mdc` files for team-specific patterns:

```yaml

description: Custom deployment workflow for our team
alwaysApply: false
---

# Custom Team Deployment

[Your team's specific patterns and guidelines]
```

## Troubleshooting

### MCP Server Not Working

1. Check your Claude Desktop configuration file
2. Verify your project ID is correct
3. Ensure you have a valid Varity API key
4. Check internet connectivity

### Cursor Rules Not Activating

1. Verify `.cursor/rules/` directory exists
2. Check file names match exactly (`.mdc` extension)
3. Restart Cursor IDE
4. Check Cursor settings for rule activation

### AI Suggestions Incorrect

1. Update to latest Cursor rules
2. Provide more context in your prompt
3. Check if using correct Varity SDK version
4. Report issues on GitHub
