<Aside type="tip">
**Most apps do not need this page.** The SDK and CLI auto-configure auth, database, storage, and deployment for you. If you are following the [Quickstart](/getting-started/quickstart/), you can skip this page entirely.
</Aside>

## When you might land here

You are building something unusual that requires direct access to low-level platform internals. For example, a custom integration that does not fit the SDK's high-level abstractions. In nearly every case, the public SDK API in [API Reference](/packages/sdk/api-reference/) covers what you need.

If you have a specific use case the SDK does not cover, [open an issue](https://github.com/varity-labs/varity-sdk/issues) and we will help you find the right entry point.

## What the SDK handles for you

| Concern | How it's handled |
|---------|------------------|
| Authentication | Managed credentials, auto-injected. Configure your own only if you must. |
| Database | Auto-provisioned per app on first deploy. |
| Storage | Auto-provisioned alongside the database. |
| Deploy URL | Allocated automatically as `varity.app/{your-app-name}/`. |
| Custom domain | See [Custom Domains](/deploy/custom-domains/). |
| Payments | Configure billing in the developer portal. |

For everything else, see [Deploy Your App](/deploy/deploy-your-app/) and [Auto-wired Services](/deploy/auto-wired-services/).
