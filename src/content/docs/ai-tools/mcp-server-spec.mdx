The Varity MCP server (`@varity-labs/mcp`) gives AI editors 8 tools for the full build ŌåÆ deploy ŌåÆ monetize workflow. It runs locally via stdio ŌĆö no API keys, no accounts.

<Aside type="tip">
The MCP server works with **any editor that supports MCP**: Cursor, Claude Code, VS Code (Copilot), Windsurf, and more. Setup takes under a minute.
</Aside>

## Prerequisites

- **Node.js 18+** ŌĆö runs the MCP server
- **Python 3.8+** ŌĆö needed for `varitykit` CLI (used by deploy tools)
- **varitykit** ŌĆö install with `pip install varitykit`

## Setup

<Tabs>
  <TabItem label="Cursor">
    Add to `.cursor/mcp.json` in your project root:

    ```json
    {
      "mcpServers": {
        "varity": {
          "command": "npx",
          "args": ["-y", "@varity-labs/mcp"]
        }
      }
    }
    ```

    Restart Cursor. The Varity tools appear in the MCP tool list.
  </TabItem>
  <TabItem label="Claude Code">
    Run once from any directory:

    ```bash
    claude mcp add varity -- npx -y @varity-labs/mcp
    ```

    Claude Code picks up the server automatically on the next conversation.
  </TabItem>
  <TabItem label="VS Code (Copilot)">
    Add to `.vscode/mcp.json`:

    ```json

        "varity": {
          "command": "npx",

        }
      }
    }
    ```
  </TabItem>
  <TabItem label="Windsurf">

## Tools Reference

The server exposes 8 tools. Read-only tools run safely with no side effects. Destructive tools create files or deploy infrastructure.

### `varity_search_docs` ŌĆö Search documentation

Search Varity docs for guides, API references, and tutorials.

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `query` | string | Yes | ŌĆö | Search query (e.g., "database setup", "how to deploy") |
| `maxResults` | number | No | 3 | Maximum results to return |

**Annotations:** `readOnlyHint: true`

**Example prompt:** "Search the Varity docs for how to set up authentication"

---

### `varity_cost_calculator` ŌĆö Estimate costs

Compare Varity hosting costs against AWS, Vercel, and other platforms.

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `users` | number | Yes | ŌĆö | Estimated monthly active users |
| `storage_gb` | number | No | 10 | Storage needed in GB |
| `has_database` | boolean | No | true | Whether the app uses a database |
| `has_auth` | boolean | No | true | Whether the app uses authentication |

**Annotations:** `readOnlyHint: true`

**Example prompt:** "How much would it cost to host a SaaS app with 500 users on Varity?"

---

### `varity_init` ŌĆö Create a new app

Scaffold a production-ready app with auth, database, and payments.

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `name` | string | Yes | ŌĆö | Project name (lowercase, hyphens allowed) |
| `template` | `"saas-starter"` | No | `"saas-starter"` | Template to use |
| `path` | string | No | Current directory | Directory to create the project in |

**Example prompt:** "Create a new Varity app called invoice-tracker"

---

### `varity_create_repo` ŌĆö Create GitHub repository

Create a new GitHub repository with the Varity SaaS template. Useful for browser-based workflows ŌĆö create a repo, open in Gitpod/StackBlitz, and deploy.

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `name` | string | Yes | ŌĆö | Repository name (lowercase, hyphens allowed) |
| `description` | string | No | ŌĆö | Short description for the repo |
| `template` | `"saas-starter"` | No | `"saas-starter"` | Template to use |
| `visibility` | `"public"` \| `"private"` | No | `"public"` | Repository visibility |
| `github_token` | string | No | ŌĆö | GitHub personal access token with `repo` scope |

**Annotations:** `destructiveHint: true` ŌĆö creates an external GitHub repository

**Example prompt:** "Create a new GitHub repo called my-dashboard with the Varity template"

---

### `varity_deploy` ŌĆö Deploy to production

Build and deploy the current project. Auto-detects framework (Next.js, React, Vue), builds it, and returns a live URL.

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `path` | string | No | Workspace root | Path to the project directory |
| `submit_to_store` | boolean | No | false | Also submit to the Varity App Store (90% revenue to you) |

**Annotations:** `destructiveHint: true` ŌĆö deploys real infrastructure

**Example prompt:** "Deploy this project to Varity"

---

### `varity_deploy_status` ŌĆö Check deployments

List all deployments or get status of a specific one. Shows URL, status, framework, size, and creation time.

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `deployment_id` | string | No | ŌĆö | Specific deployment ID (omit to list all) |

**Annotations:** `readOnlyHint: true`

**Example prompt:** "Show me the status of my deployments"

---

### `varity_deploy_logs` ŌĆö Get build logs

Get build and deployment logs for debugging failed or recent deployments.

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `deployment_id` | string | Yes | ŌĆö | The deployment ID to get logs for |
| `limit` | number | No | 100 | Maximum log lines to return |

**Annotations:** `readOnlyHint: true`

**Example prompt:** "Show me the build logs for deployment abc123"

---

### `varity_submit_to_store` ŌĆö Publish to App Store

Submit a deployed app to the Varity App Store marketplace. Revenue split: 90% to you, 10% to Varity.

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `deployment_id` | string | Yes | ŌĆö | Deployment ID from a previous `varity_deploy` |
| `name` | string | Yes | ŌĆö | Display name on the store |
| `description` | string | Yes | ŌĆö | Short description shown to users |
| `price` | number | Yes | ŌĆö | Monthly price in USD (use 0 for free apps) |

**Annotations:** `destructiveHint: true` ŌĆö publishes to a public store

**Example prompt:** "Submit my last deployment to the Varity App Store at $9.99/month"

## Typical Workflow

Here's the full build-to-monetize flow using the MCP tools:

1. **"Create a Varity app called my-saas"** ŌåÆ `varity_init` (local) or `varity_create_repo` (GitHub)
2. **"What will this cost for 200 users?"** ŌåÆ `varity_cost_calculator`
3. *(Build your features)*
4. **"Deploy this"** ŌåÆ `varity_deploy`
5. **"Did the deploy succeed?"** ŌåÆ `varity_deploy_status`
6. **"Show me the build logs"** ŌåÆ `varity_deploy_logs`
7. **"Submit to the app store at $14.99"** ŌåÆ `varity_submit_to_store`

## Troubleshooting

**"Cannot find module @varity-labs/mcp"**
Make sure Node.js 18+ is installed. Run `node --version` to check.

**Deploy tools fail with "varitykit not found"**
Install the CLI: `pip install varitykit`. Run `varitykit doctor` to verify.

**Server not showing in Cursor**
Restart Cursor after adding the config. Check that `.cursor/mcp.json` is valid JSON.

**Server not showing in Claude Code**
Run `claude mcp list` to verify the server is registered. Re-add with `claude mcp add varity -- npx -y @varity-labs/mcp`.

## HTTP Transport (Claude.ai, ChatGPT, Browser-Based LLMs)

The MCP server supports two transports:

- **stdio** (default) ŌĆö for desktop editors like Cursor, Claude Code, VS Code, Windsurf
- **HTTP** ŌĆö for browser-based AI tools like Claude.ai and ChatGPT

### Hosted MCP

Connect directly to the hosted Varity MCP at:

```
https://mcp.varity.so
```

**Claude.ai:** Settings ŌåÆ Connectors ŌåÆ Add MCP Server ŌåÆ URL: `https://mcp.varity.so`

**ChatGPT:** Settings ŌåÆ Connectors ŌåÆ Create ŌåÆ MCP server URL: `https://mcp.varity.so`

The first time you connect, you'll authenticate via the Varity login page.

### Self-Hosted HTTP

Run the HTTP server locally for development:

```bash
npx -y @varity-labs/mcp --transport http --port 3100
```

This starts the MCP server at `http://localhost:3100/mcp`.

<Aside type="note">
`varity_init` (scaffolding) is only available via stdio transport since it requires access to the local filesystem. All other tools work on both transports.
</Aside>
