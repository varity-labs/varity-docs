VarityKit is a Python-based CLI that helps you deploy and manage applications. If you have used tools like the Vercel CLI, Netlify CLI, or Firebase CLI, you will feel right at home.

## What You Can Do

<CardGrid>
  <Card title="Deploy Applications" icon="rocket">
    Deploy any Node or Python app with one command. Static sites and dynamic apps both supported.
  </Card>
  <Card title="Migrate from Vercel" icon="seti:config">
    Move an existing Vercel app to Varity in one step: analyze, transform, and deploy.
  </Card>
  <Card title="Manage Deployments" icon="list-format">
    List deployments, view status and logs, and roll back to a previous version.
  </Card>
  <Card title="Validate Setup" icon="approve-check">
    Run diagnostics to ensure your development environment is ready.
  </Card>
</CardGrid>

## Commands at a Glance

### Core Commands

| Command | Description |
|---------|-------------|
| [`varitykit login`](/cli/commands/auth) | Log in with your deploy key from the developer portal |
| [`varitykit doctor`](/cli/commands/doctor) | Validate your development environment |
| [`varitykit dev`](#dev) | Start development servers (API + frontend) |
| [`varitykit bootstrap`](#bootstrap) | Install project dependencies and set up environment |
| [`varitykit completions`](#completions) | Set up shell completions (bash, zsh, fish) |

### Authentication

| Command | Description |
|---------|-------------|
| [`varitykit auth login`](/cli/commands/auth) | Log in with your deploy key |
| [`varitykit auth logout`](/cli/commands/auth#logout) | Remove saved credentials |
| [`varitykit auth status`](/cli/commands/auth#status) | Show current login status |

### App Deployment

| Command | Description |
|---------|-------------|
| [`varitykit app deploy`](/cli/commands/deploy) | Deploy your application |
| `varitykit app list` | List all deployments |
| `varitykit app info` | Show deployment details |
| `varitykit app status` | Show deployment status |
| `varitykit app rollback` | Rollback to a previous deployment |

### Domains

| Command | Description |
|---------|-------------|
| `varitykit domains list` | List all domains you own on `varity.app` |

### Migration (Vercel → Varity)

| Command | Description |
|---------|-------------|
| `varitykit migrate --url <github-url>` | Clone a GitHub repo and migrate it to Varity (analyze + transform + deploy) |
| `varitykit migrate --path <path>` | Migrate a local Vercel project directory |
| `varitykit migrate --dry-run` | Preview what would change without modifying files |
| `varitykit migrate --no-deploy` | Apply codemods only, skip deployment |
| `varitykit migrate analyze <path>` | Scan for Vercel artifacts (read-only preview) |
| `varitykit migrate apply <path>` | Apply codemods without deploying |
| `varitykit migrate rollback <path>` | Restore files from `.vercel-migration-backup/` |

## Quick Start

```bash
# Install the CLI
pipx install varitykit

# Check your environment
varitykit doctor

# Log in with your developer portal deploy key
varitykit login

# Deploy (auto-detects static vs dynamic)
varitykit app deploy

# Or force a specific mode
varitykit app deploy --hosting dynamic
varitykit app deploy --hosting static
```

## Global Options

All commands support these global options:

```bash
varitykit [OPTIONS] COMMAND [ARGS]...

Options:
  --version        Show the CLI version
  -v, --verbose    Enable verbose output
  --debug          Enable debug output
  --json           Output in JSON format
  --help           Show help message
```

## Requirements

- **Python**: 3.8 or higher
- **Node.js**: 20+ (for building JavaScript projects)

</Aside>

## Hosting Options

### Static Sites

Deploy static websites and single-page applications:

```bash
varitykit app deploy
```

- **Best for**: Marketing sites, documentation, SPAs
- **Cost**: One flat monthly cost per app, starting around $5/mo
- **Speed**: Global CDN distribution

### Dynamic Apps

Deploy full-stack applications with backend:

```bash
varitykit app deploy --hosting dynamic
```

- **Best for**: APIs, databases, server-side rendering, AI agents, LLMs
- **Cost**: One flat monthly cost per app, based on the hardware you reserve
- **Features**: Containers, scaling, persistence

The orchestration algorithm auto-detects when your app needs dynamic hosting (Express, Fastify, Next.js SSR, FastAPI, Django, Flask, etc.) and selects it automatically. Use `--hosting dynamic` only to force the mode explicitly.

## Supported Frameworks

Varity currently auto-detects and deploys the following frameworks:

| Framework | Static | Dynamic |
|-----------|--------|---------|
| Next.js 13+ | ✅ | ✅ |
| React (Vite) | ✅ | ✅ |
| React (CRA) | ✅ | ✅ |
| Vue 3+ | ✅ | ✅ |
| Express, Fastify, Nest, Koa, Hono | N/A | ✅ |
| FastAPI, Django, Flask | N/A | ✅ |

**Not yet supported:** Go, Rust, Ruby / Rails, Elixir / Phoenix, Java / Spring, Deno, PHP / Laravel, .NET. These will not be auto-detected and deployment will fail with an unsupported-language error. [Request support for your framework](https://github.com/varity-labs).

## Additional Command Details

### dev

Start development servers (API + frontend) with hot-reload:

```bash
varitykit dev [OPTIONS]
```

| Option | Default | Description |
|--------|---------|-------------|
| `--project` | `marketing` | Frontend project to run |
| `--api-only` | `false` | Run only the API server |
| `--frontend-only` | `false` | Run only the frontend |
| `--port` | `3001` | API server port |

### bootstrap

Install project dependencies and set up the environment:

```bash
varitykit bootstrap [OPTIONS]
```

| Option | Description |
|--------|-------------|
| `--skip-npm` | Skip npm install |
| `--skip-pip` | Skip pip install |
| `--skip-docker` | Skip Docker setup |

### completions

Set up shell completions for the CLI:

```bash
varitykit completions [OPTIONS]
```

| Option | Description |
|--------|-------------|
| `--shell` | Shell type: `bash`, `zsh`, or `fish` (auto-detected if omitted) |
| `--install` | Install completions to your shell config file |
