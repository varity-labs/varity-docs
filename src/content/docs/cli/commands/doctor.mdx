Validate your development environment and identify any issues.

## Usage

```bash
varitykit doctor [OPTIONS]
```

## What It Checks

The doctor command performs 20+ checks:

### System Requirements

| Check | Requirement |
|-------|-------------|
| Python version | >= 3.8 |
| Node.js version | >= 18.0 |
| npm version | Any |
| Docker | Running (optional) |

### Environment Variables

| Variable | Status |
|----------|--------|
| `THIRDWEB_CLIENT_ID` | Required for IPFS deployments |
| `AKASH_WALLET_KEY` | Required for Akash deployments |
| `WALLET_PRIVATE_KEY` | Required for contract deployments |

### Network Connectivity

- Varity L3 RPC connection
- thirdweb API access
- IPFS gateway access

### Project Structure

- `package.json` exists
- Build script defined
- Dependencies installed

## Options

| Option | Description |
|--------|-------------|
| `--verbose` | Show detailed output for each check |
| `--json` | Output results as JSON |
| `--fix` | Attempt to fix common issues |

## Exit Codes

| Code | Meaning |
|------|---------|
| 0 | All checks passed |
| 1 | Some checks failed |
| 2 | Critical failure (cannot proceed) |

## Fixing Common Issues

### Missing Node.js

```bash
# macOS
brew install node

# Ubuntu
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Windows
# Download from nodejs.org
```

### Missing Environment Variables

Create a `.env` file in your project root:

```bash title=".env"
THIRDWEB_CLIENT_ID=your_client_id
AKASH_WALLET_KEY=your_wallet_key
WALLET_PRIVATE_KEY=your_private_key
```

### Docker Not Running

```bash
# macOS/Windows
# Open Docker Desktop

# Linux
sudo systemctl start docker
```

<Aside type="tip">
Run `varitykit doctor --fix` to automatically fix common issues like missing directories or configuration files.
</Aside>
