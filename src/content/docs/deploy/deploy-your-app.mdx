:::

## Three Ways In

Varity accepts three kinds of input, from the CLI, the Developer Portal, or the MCP server in your AI coding tool:

| Input | How | Best for |
|-------|-----|----------|
| **Your project / GitHub repo** | `varitykit app deploy` (this page) | Apps in a [supported framework](/deploy/supported-frameworks) — auto-detected, auto-built |
| **A Docker image** | `varitykit app deploy --image <ref>` — see [Deploy a Docker Image](/deploy/deploy-docker-image) | Anything already containerized, any language |
| **A template** | `varitykit app deploy --agent <name>` or the portal gallery — see [agent templates](/tutorials/deploy-ai-agent#one-click-agent-templates) | One-click AI agent and tool deployments |

The rest of this page covers the first path: deploying your own project.

## Quick Deploy

<Steps>

1. **Install the CLI**

   ```bash
   pipx install varitykit
   ```

2. **Build your project**

   ```bash
   npm run build
   ```

3. **Deploy**

   ```bash
   varitykit app deploy
   ```

4. **Your app is live!**

   Varity will output your live URL:

   ```
   ✓ Deployed successfully!
   ✓ URL: https://your-app.varity.app
   ✓ Build time: 1m 23s
   ✓ Status: Live
   ```

</Steps>

## What Happens During Deployment

When you run `varitykit app deploy`:

1. **Detects your framework** - Automatically identifies Next.js, React, Vue, etc.
2. **Builds your app** - Runs your build command (`npm run build`)
3. **Uploads your files** - Stores them securely with automatic backups
4. **Makes your app live** - Generates a public URL
5. **Returns a live URL** - Static apps use `https://varity.app/<name>/`; dynamic apps use `https://<name>.varity.app/`

All of this happens automatically.

## Before You Deploy

### Environment Variables

Credentials are managed automatically by Varity. Optionally set your app ID:

```bash title=".env.local"
# Optional: Your Varity App ID
NEXT_PUBLIC_VARITY_APP_ID=<your-app-id>

```

See [Environment Variables](/deploy/env-variables) for the complete list and [Managed Credentials](/deploy/managed-credentials) for how credential auto-provision works.

### Test Locally First

Always test your build works locally:

```bash
npm run build
npm run start
```

Visit `http://localhost:3000` and verify everything works.

## Manage Your Deployments

### List All Deployments

View your deployment history:

```bash
varitykit app list
```

Output:
```
ID                  Status    URL                                Deployed
deploy-1737491000   live      https://your-app.varity.app       2 hours ago
deploy-1737484800   archived  https://your-app-v2.varity.app    1 day ago
deploy-1737398400   archived  https://your-app-v1.varity.app    2 days ago
```

### Rollback to Previous Version

Revert to a previous deployment:

```bash
varitykit app rollback deploy-1737484800
```

Your app will instantly switch to the selected version. See [Rollback a Deployment](/deploy/rollback) for a full walkthrough, including how to find deployment IDs and handle caveats around environment variables and database migrations.

### View Deployment Details

Check the details of a specific deployment:

```bash
varitykit app info deploy-1737491000
```

### Check Deployment Status

View overall deployment status:

```bash
varitykit app status
```

## Deployment URL

Every deployment gets a public URL. URL format depends on hosting type:

```bash
varitykit app deploy --name my-app
# Static hosting → https://varity.app/my-app/
# Dynamic hosting → https://my-app.varity.app/
```

See [Custom Domains](/deploy/custom-domains) for the current status of bring-your-own domains.

:::note[Bring your own domain]
Pointing `myapp.com` at a Varity deployment via DNS is under development. See [Custom Domains](/deploy/custom-domains) for status.
:::

## Troubleshooting

### "Deployment failed: Build error"

1. Verify your build works locally:
   ```bash
   npm run build
   ```

2. Check the build output directory exists:
   - Next.js: `.next/` or `out/`
   - React: `dist/` or `build/`
   - Vue: `dist/`

### "Environment variables not found"

Credentials are auto-provided. If you see this error, run `varitykit doctor` to check your setup.

### "Deployment timed out"

Check your internet connection:

```bash
varitykit doctor
```

If issues persist, try deploying with verbose output:

```bash
varitykit app deploy --verbose
```
