Install the Varity tools that fit how you work. The MCP server is the primary path for most developers using Claude Code or Cursor.

## System Requirements

- **Node.js** 20.0 or later
- **Python** 3.11 or later (for the CLI)
- **npm**, **pnpm**, or **yarn**
- A modern browser (Chrome, Firefox, Safari, Edge)

---

## Option 1: MCP Server (recommended for Claude Code and Cursor)

The MCP server connects Varity to your AI coding tool. It gives you 17 tools for deploying, monitoring, and managing apps using plain English. You do not need to add any code to your project.

<Tabs>
  <TabItem label="Claude Code">
    ```bash
    claude mcp add varity -- npx -y @varity-labs/mcp
    ```

    Restart Claude Code after running this command. You will see Varity tools available in your session.
  </TabItem>
  <TabItem label="Cursor">
    Add the following to your Cursor MCP config file (`~/.cursor/mcp.json`):

    ```json
    {
      "mcpServers": {
        "varity": {
          "command": "npx",
          "args": ["-y", "@varity-labs/mcp"]
        }
      }
    }
    ```

    Restart Cursor after saving the file.
  </TabItem>
  <TabItem label="Other AI tools">
    For any MCP-compatible AI tool, use:

    ```json
    {
      "command": "npx",
      "args": ["-y", "@varity-labs/mcp"]
    }
    ```

    Refer to your AI tool's documentation for how to add MCP servers.
  </TabItem>
</Tabs>

<Aside type="tip">
Once the MCP server is installed, you do not need to add any code to your app to deploy it. The MCP handles the full deploy flow automatically.
</Aside>

### Verify MCP installation

Inside Claude Code or Cursor, ask:

```
varity_doctor
```

You should see a confirmation that your environment is ready.

## Option 2: CLI (command line)

The CLI is a Python package called `varitykit`. If you are a JavaScript developer, do not let that put you off. Python is only needed to run the CLI itself, not your app. You install it once and never think about it again.

<Steps>

1. **Install the CLI**

   The recommended way is with `pipx`, which installs CLI tools in their own isolated environment:

   ```bash
   pipx install varitykit
   ```

   If you do not have `pipx`, you can install it first:

   ```bash
   pip install pipx
   ```

   Then run:

   ```bash
   pipx ensurepath
   ```

   Or use plain `pip` if you prefer:

   ```bash
   pip install varitykit
   ```

2. **Verify installation**

   ```bash
   varitykit doctor
   ```

   You should see all checks passing.

3. **Log in**

   ```bash
   varitykit login
   ```

   This opens the developer portal in your browser. Copy your deploy key and paste it back into the terminal when prompted.

</Steps>

<Aside type="note" title="Windows users">
If `varitykit` is not found after installing with `pip` on Windows, your Python Scripts folder may not be in your PATH. Using `pipx install varitykit` avoids this issue entirely.
</Aside>

## Option 3: npm Packages (optional, for SDK features)

The npm packages are optional. You only need them if you want to use Varity SDK features in your app code (like auth hooks or UI components). You can deploy any app without them.

<CardGrid>
  <Card title="@varity-labs/sdk">
    Auth, storage, and database helpers for your app code.

    ```bash
    npm install @varity-labs/sdk
    ```
  </Card>
  <Card title="@varity-labs/ui-kit">
    React components for login, dashboards, and account management.

    ```bash
    npm install @varity-labs/ui-kit
    ```
  </Card>
  <Card title="@varity-labs/types">
    TypeScript type definitions for all Varity interfaces.

    ```bash
    npm install @varity-labs/types
    ```
  </Card>
</CardGrid>

Install all three at once:

<Tabs>
  <TabItem label="npm">
    ```bash
    npm install @varity-labs/sdk @varity-labs/ui-kit @varity-labs/types
    ```
  </TabItem>
  <TabItem label="pnpm">
    ```bash
    pnpm add @varity-labs/sdk @varity-labs/ui-kit @varity-labs/types
    ```
  </TabItem>
  <TabItem label="yarn">
    ```bash
    yarn add @varity-labs/sdk @varity-labs/ui-kit @varity-labs/types
    ```
  </TabItem>
</Tabs>

## No framework changes required

You do not need to wrap your app in any provider component or change your existing code to deploy. Varity detects your framework and deploys it as-is.

If you choose to use the `@varity-labs/sdk` or `@varity-labs/ui-kit` packages for auth or UI features, the SDK reference docs show how to add them.

## Troubleshooting

### CLI not found after installation

If `varitykit` is not recognized after `pip install`, try:

```bash
python -m varitykit doctor
```

Or reinstall with `pipx`:

```bash
pipx install varitykit
```

### Node.js version errors

Varity requires Node.js 20 or later. Check your version:

```bash
node --version
```

If you need to upgrade, use [nvm](https://github.com/nvm-sh/nvm) or download from [nodejs.org](https://nodejs.org).

### TypeScript errors after installing SDK packages

Make sure you are using TypeScript 5.0 or later:

```bash
npm install typescript@latest
```

For more help, see [Troubleshooting](/resources/troubleshooting) or join our [Discord](https://discord.gg/7vWsdwa2Bg).
