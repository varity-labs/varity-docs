# Environment Variables

Configure environment variables for your Varity application.

## Required Variables

### THIRDWEB_CLIENT_ID

Required for all deployments. Used for IPFS uploads and blockchain interactions.

```bash title=".env"
THIRDWEB_CLIENT_ID=your_thirdweb_client_id
```

**How to get it:**
1. Go to [thirdweb Dashboard](https://thirdweb.com/dashboard)
2. Create or select a project
3. Copy your Client ID

### AKASH_WALLET_KEY (Akash deployments only)

Required for deploying dynamic applications to Akash.

```bash title=".env"
AKASH_WALLET_KEY=your_akash_wallet_private_key
```

<Aside type="caution">
Never commit your `AKASH_WALLET_KEY` to version control. Use environment variables or a secrets manager.
</Aside>

## Application Variables

### Authentication (Privy)

```bash title=".env"
# Privy App ID - get from https://dashboard.privy.io
NEXT_PUBLIC_PRIVY_APP_ID=your_privy_app_id

# Or use Varity development credentials (for testing only)
NEXT_PUBLIC_VARITY_APP_ID=varity_dev_app_id
```

### Varity SDK

```bash title=".env"
# Optional: Override default Varity API endpoint
VARITY_API_URL=https://api.varity.io

# Optional: Your Varity API key
VARITY_API_KEY=your_api_key

# Optional: Your wallet address
VARITY_WALLET_ADDRESS=0x...
```

## Framework-Specific Configuration

### Next.js

Next.js requires the `NEXT_PUBLIC_` prefix for client-side variables:

```bash title=".env.local"
# Client-side (exposed to browser)
NEXT_PUBLIC_PRIVY_APP_ID=your_app_id
NEXT_PUBLIC_THIRDWEB_CLIENT_ID=your_client_id

# Server-side only
THIRDWEB_SECRET_KEY=your_secret_key
```

### Vite (React, Vue)

Vite requires the `VITE_` prefix for client-side variables:

```bash title=".env"
# Client-side (exposed to browser)
VITE_PRIVY_APP_ID=your_app_id
VITE_THIRDWEB_CLIENT_ID=your_client_id
```

### Create React App

CRA requires the `REACT_APP_` prefix:

```bash title=".env"
# Client-side (exposed to browser)
REACT_APP_PRIVY_APP_ID=your_app_id
REACT_APP_THIRDWEB_CLIENT_ID=your_client_id
```

## Security Best Practices

<Aside type="danger">
**Never expose secret keys in client-side code.** Only use `_PUBLIC_` prefixed variables for client-side access.
</Aside>

### Do:
- Use `.env.local` for local development (git-ignored)
- Store production secrets in your CI/CD platform
- Use different API keys for development and production
- Rotate keys regularly

### Don't:
- Commit `.env` files with real credentials
- Use production keys in development
- Share API keys in public repositories
- Hardcode secrets in source code

## Verifying Configuration

Run the doctor command to verify your configuration:

```bash
varitykit doctor
```

This checks:
- Required environment variables
- Network connectivity
- API key validity
- Project structure

## Troubleshooting

### "THIRDWEB_CLIENT_ID not set"

1. Create a `.env` file in your project root
2. Add `THIRDWEB_CLIENT_ID=your_client_id`
3. Restart your terminal or run `source .env`

### Variables not loading

1. Check file name is exactly `.env` or `.env.local`
2. Restart your development server after changes
3. Verify no syntax errors in the file
4. Check framework-specific prefixes (`NEXT_PUBLIC_`, `VITE_`, etc.)
