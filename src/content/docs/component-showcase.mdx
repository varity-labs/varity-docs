## 2. Callout Component

Four semantic types for different message priorities.

<Callout type="tip" title="Pro tip">
  Use environment variables for all API keys and sensitive configuration.
  Never commit credentials to Git.
</Callout>

<Callout type="note">
  The default RPC endpoint is `https://rpc-varity-testnet.t.conduit.xyz`.
  You can override this in your Varity SDK configuration.
</Callout>

<Callout type="warning" title="Breaking change in v2.0">
  The `createClient()` function signature has changed. Update your code
  to use the new format: `createClient(url, apiKey, options)`.
</Callout>

<Callout type="danger" title="Security warning">
  Never expose your service role key in client-side code. This key has
  full access to your Varity project and should only be used in server
  environments.
</Callout>

### Collapsible Callout

<Callout type="tip" title="Advanced usage" collapsible={true} defaultOpen={false}>
  You can pass additional options to customize the SDK behavior:

  ```typescript
  const varity = createClient(url, key, {
    network: 'ethereum',
    timeout: 5000,
    retries: 3
  })
  ```

  See the [SDK Configuration Guide](/docs/packages/sdk/configuration) for all options.
</Callout>

### Advanced Variant (Layer 2 Content)

<Accordion title="Advanced: How smart accounts work" variant="advanced">
  Varity uses ERC-4337 Account Abstraction for smart accounts. When you create
  an account:

  1. **Factory** creates a new smart account at a deterministic address
  2. **Session keys** enable free operations for specific actions
  3. **Paymaster** (`0xeF467aef91d4e626C7e56967779069bEF22c4453`) sponsors usage fees
  4. **User operations** are bundled and executed efficiently

  ```typescript
  // Create a smart account
  const account = await varity.accounts.create({
    owner: privyUser.wallet.address
  })

  // Grant session key for specific action
  await account.grantSessionKey({
    target: '0x...',
    selector: '0x...',
    validUntil: Date.now() + 3600000 // 1 hour
  })
  ```

  [Learn more about smart accounts â†’](/docs/build/wallets/quickstart)
</Accordion>

### Example Variant

<Accordion title="Example: Complete authentication flow" variant="example" defaultOpen={true}>
  Here's a complete example showing email login, session handling, and logout:

  ```typescript
  import { createClient } from '@varity-labs/sdk'

  const varity = createClient(
    process.env.VARITY_URL!,
    process.env.VARITY_KEY!
  )

  // Login
  async function login(email: string, password: string) {
    const { data, error } = await varity.auth.login({ email, password })

    if (error) {
      throw new Error(`Login failed: ${error.message}`)
    }

    // Session is automatically stored in cookies
    console.log('User:', data.user.email)
    console.log('Session expires:', data.session.expiresAt)

    return data
  }

  // Check session
  async function getUser() {
    const { data, error } = await varity.auth.getUser()

    if (error || !data) {
      return null
    }

    return data.user
  }

  // Logout
  async function logout() {
    await varity.auth.logout()
    console.log('Logged out')
  }

  // Usage
  await login('user@example.com', 'secure-password')
  const user = await getUser()
  console.log('Current user:', user?.email)
  await logout()
  ```
</Accordion>
