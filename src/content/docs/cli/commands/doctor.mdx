Validate your development environment and identify any issues.

## Usage

```bash
varitykit doctor [OPTIONS]
```

## What It Checks

The doctor command runs 8 checks across 4 categories:

### Required Tools

| Check | Requirement |
|-------|-------------|
| Node.js | >= 16.0.0 |
| npm | Any version |
| git | Any version |

### System Resources

| Check | Requirement |
|-------|-------------|
| Disk Space | 2GB free |
| Memory | 3GB available |

<Aside type="note">
The Memory check requires 3GB available RAM. On WSL, CI machines, or low-memory VMs this check will often fail. It is a warning — your deploy will still succeed as long as builds complete locally.
</Aside>

### Project Configuration

| Check | Requirement |
|-------|-------------|
| `varity.config.json` | Present (warns if missing — not required for all deploy paths) |
| `.env` file | Present (optional — warns if absent) |

### Network

| Check | Requirement |
|-------|-------------|
| Internet | Reachable |

## Options

| Option | Description |
|--------|-------------|
| `--fix` | Attempt to auto-fix common issues |
| `--full` | Run full diagnostics including all infrastructure checks |

<Aside type="note">
`--verbose` is a global option available on all `varitykit` commands.
</Aside>

## Exit Codes

| Code | Meaning |
|------|---------|
| 0 | All checks passed |
| 1 | Some checks failed |
| 2 | Critical failure (cannot proceed) |

## Fixing Common Issues

### Missing Node.js

```bash
# macOS
brew install node

# Ubuntu
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Windows
# Download from nodejs.org
```

### Low Memory Warning

If the Memory check reports a warning (requires 3GB available), you can safely continue — the warning means builds may be slower on this machine, but deployments still work.

On WSL, run `wsl --shutdown` and reopen to reclaim memory from the host OS if needed.

<Aside type="tip">
Run `varitykit doctor --fix` to automatically fix common issues like missing directories or configuration files.
</Aside>

## Verbose Output

For more detailed diagnostics:

```bash
varitykit doctor --full
```

This runs all infrastructure checks in addition to the standard environment checks.
