Create a new Varity project from a starter template.

## Usage

```bash
varitykit init [PROJECT_NAME] [OPTIONS]
```

## Quick Start

```bash
# Interactive mode
varitykit init

# With project name
varitykit init my-app

# With template
varitykit init my-app --template saas-starter
```

## Available Templates

| Template | Description | Best For |
|----------|-------------|----------|
| `saas-starter` | SaaS dashboard with auth, analytics, and billing | Most applications |

<Aside type="tip">
The `saas-starter` template includes everything you need: authentication, dashboard layout, analytics, and deployment-ready configuration.
</Aside>

## Options

| Option | Default | Description |
|--------|---------|-------------|
| `--template` | `saas-starter` | Template to use |
| `--skip-install` | `false` | Skip `npm install` |
| `--skip-git` | `false` | Skip git initialization |
| `--typescript` | `true` | Use TypeScript |
| `--package-manager` | `npm` | npm, pnpm, or yarn |

## Interactive Mode


## Project Structure

The generated project includes:

```
my-app/
├── .env.local.example    # Environment variables template
├── .gitignore
├── package.json
├── tsconfig.json
├── next.config.mjs
├── src/
│   ├── app/
│   │   ├── layout.tsx    # Root layout with PrivyStack
│   │   ├── page.tsx      # Home page
│   │   └── dashboard/
│   │       └── page.tsx  # Protected dashboard
│   ├── components/
│   │   ├── Header.tsx
│   │   └── LoginButton.tsx
│   └── lib/
│       └── varity.ts     # Varity SDK setup
└── public/
    └── logo.svg
```

## What's Included

The SaaS starter template includes:

- **Authentication** - PrivyStack pre-configured
- **Protected Routes** - PrivyProtectedRoute examples
- **Dashboard Layout** - Responsive sidebar and navigation
- **Analytics** - Data tables, charts, and KPIs
- **Environment Variables** - `.env.local.example` with all variables
- **TypeScript** - Full type support
- **Styling** - Tailwind CSS configured
- **Deployment Ready** - Works with `varitykit app deploy`

## Environment Setup

After initialization, set up your environment:

<Steps>

1. **Copy the example environment file**

   ```bash
   cp .env.local.example .env.local
   ```

2. **Start development server**

   ```bash
   npm run dev
   ```

</Steps>

<Aside type="tip">
No credentials needed. Varity manages all credentials automatically. See [Managed Credentials](/deploy/managed-credentials).
</Aside>

## Custom Templates

Create your own template:

```bash
# Save current project as template
varitykit template save my-template

# Use custom template
varitykit init new-project --template my-template

# List available templates
varitykit template list
```

## Troubleshooting

### "Template not found"

List available templates:

```bash
varitykit template list
```

### "npm install failed"

Try with a different package manager:

```bash
varitykit init my-app --package-manager pnpm
```

Or skip install and do it manually:

```bash
varitykit init my-app --skip-install
cd my-app
npm install
```
