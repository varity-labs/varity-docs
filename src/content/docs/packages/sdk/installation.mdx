Get the Varity SDK installed and configured in your project.

## Install the Package

<Tabs>
  <TabItem label="npm">
    ```bash
    npm install @varity-labs/sdk
    ```
  </TabItem>
  <TabItem label="pnpm">
    ```bash
    pnpm add @varity-labs/sdk
    ```
  </TabItem>
  <TabItem label="yarn">
    ```bash
    yarn add @varity-labs/sdk
    ```
  </TabItem>
</Tabs>

## Quick Setup

<Steps>

1. **Import and initialize the SDK**

   ```typescript
   import { createVaritySDK } from '@varity-labs/sdk';

   const sdk = createVaritySDK();
   // Network is configured automatically
   ```

2. **Connect to the network**

   ```typescript
   await sdk.connect();
   ```

3. **Start using SDK features**

   ```typescript
   // Upload a file
   const result = await sdk.storage.upload(fileData);

   // Get analytics
   const kpis = await sdk.analytics.getKPIs({ period: 'current_month' });
   ```

</Steps>

## Credentials

Varity manages all credentials automatically. No environment variables are needed to start developing.

<Aside type="tip">
The SDK uses shared development credentials by default. When you deploy with `varitykit deploy`, production credentials are provided automatically. See [Managed Credentials](/deploy/managed-credentials) for details.
</Aside>

## TypeScript Configuration

The SDK is written in TypeScript. Recommended `tsconfig.json` settings:

```json title="tsconfig.json"
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "esModuleInterop": true
  }
}
```

## Bundle Size

The SDK supports tree-shaking. Import only what you need.

```typescript
// Good - imports only what's needed
import { createVaritySDK, varityL3Testnet } from '@varity-labs/sdk';

// Avoid - imports the entire package
import * as VaritySDK from '@varity-labs/sdk';
```

Approximate bundle sizes:
- Full SDK: ~50KB gzipped
- Core only: ~20KB gzipped
